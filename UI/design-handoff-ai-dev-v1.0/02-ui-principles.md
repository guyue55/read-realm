# 02 UI 原则

## 阅读 UI 总原则

阅读 UI 的目标是让用户忘记 UI，而不是展示功能堆叠。

```text
低存在感 / 低干扰 / 低疲劳 / 高可控 / 强恢复
```

## 三层结构

| 层级       | 目标           | 典型组件                                                   |
| ---------- | -------------- | ---------------------------------------------------------- |
| 沉浸阅读层 | 长时间阅读舒服 | ReaderCanvas、正文排版、背景、亮度                         |
| 操作控制层 | 需要时快速操作 | TopBar、BottomBar、TocDrawer、SettingsSheet、ProgressSheet |
| 内容增强层 | 小说专属辅助   | QualityBadge、AIReaderPanel、章节净化、前情提要            |

## 视觉关键词

应接近：

```text
安静 / 柔和 / 纸感 / 稳定 / 沉浸 / 可控 / 可恢复 / 圆润
```

不应接近：

```text
资讯流 / 电商首页 / 后台系统 / 游戏大厅 / 社交 App / 强运营平台
```

## 禁止跑偏

1. 阅读页常驻复杂工具栏。
2. 阅读页出现推荐流、广告位、大运营卡片。
3. 桌面正文铺满宽屏。
4. 移动端照搬桌面三栏布局。
5. 设置面板、目录、AI 面板同时常驻移动端。
6. 纯黑 + 纯白作为默认夜间模式。
7. 尖锐硬边、大面积强色块、过重阴影。
8. 直接把 SVG 嵌入页面当 UI。
