# 图片与 SVG 颜色 / 背景差异核查

> 本次仅核查，不修改设计。

## 结论

当前源 PNG 与随包真矢量 SVG 在大方向上保持同一套纸感、米白、浅绿、低对比夜间的设计语言；但二者不是逐像素一致。

主要差异来自：

1. PNG 是生图输出，带有纸纹、柔光、微弱渐变、抗锯齿噪声和封面图片细节。
2. 真矢量 SVG 是结构化复刻，背景和卡片多为纯色或简单渐变，更干净、更平整。
3. SVG 为了可编辑性弱化了部分封面纹理、阴影细节和复杂图片质感。

## 抽样结果

- 背景 RGB 平均差异：约 `5.5` / 255
- 全图 RGB 平均差异：约 `7.0` / 255
- 最大背景抽样差异：约 `20.3` / 255

判断：**有差异，但主要是质感与细节层面的差异，不是颜色体系或背景方向完全跑偏。** 视觉上 PNG 更有生图质感，SVG 更适合开发交付、组件拆分和 Figma/Illustrator 二次编辑。

## 对开发的影响

- 前端实现不要按 PNG 逐像素复刻。
- 前端颜色、背景、圆角、字号、行高、正文宽度应以 `tokens/readerTokens.*` 为准。
- 如果需要追求最终视觉一致性，应以最新设计基线重新校准 SVG token，而不是把 PNG 当图片贴进页面。

详细数据见：`source-boards/image_svg_color_comparison.json`。
