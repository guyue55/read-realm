# 06 交互状态规范

## 移动端阅读点击区

```text
左 25%：上一页 / 上一屏
中 50%：显示 / 隐藏菜单
右 25%：下一页 / 下一屏
```

要求：

- 点击区由 `TapZoneLayer` 实现，不污染正文 DOM。
- 阅读菜单 3 秒无操作后自动淡出。
- 翻页/滚动与选中文本、长按、系统菜单不冲突。
- 对三栏桌面阅读，点击区必须按阅读容器 `getBoundingClientRect()` 的局部坐标计算。

## 阅读菜单态

- 顶部临时栏：返回、书名、更多。
- 底部菜单：目录、进度、设置、夜间、书签、AI。
- 不出现全局底部主导航。
- 菜单不得遮挡正文关键内容。
- 安全区必须适配 `env(safe-area-inset-bottom)`。

## 设置面板

- 移动端使用半屏底部 Sheet。
- Web 使用右侧柔和面板或浮层。
- 字号、行高、边距、背景修改后正文实时生效。
- 面板关闭后回到沉浸阅读层。

## 目录抽屉

- 当前章必须高亮。
- 异常章用 QualityBadge 标识。
- 支持搜索章节。
- 跳章后关闭抽屉并恢复阅读。

## AI 面板

- AI 是内容增强层，不是悬浮聊天机器人。
- 默认不抢正文焦点。
- 输出包括章节摘要、前情提要、人物关系、术语解释。
- AI 结果作为派生视图，不覆盖原文。
