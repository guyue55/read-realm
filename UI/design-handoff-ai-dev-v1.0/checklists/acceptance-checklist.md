# 08 验收清单

## A. 全局验收

- [ ] 所有页面使用 Design Tokens，不出现随意色值和尺寸。
- [ ] SVG 只作为视觉参考，不直接嵌入最终 UI。
- [ ] 页面支持真实数据渲染，不是静态 HTML 截图。
- [ ] 空、错、加载、离线、弱网状态都有明确入口。
- [ ] 移动端、桌面端、PWA 三类场景均可用。

## B. 阅读页验收

- [ ] 阅读页默认沉浸，无常驻复杂工具栏。
- [ ] 移动端阅读页无底部主导航。
- [ ] 正文最大宽度符合 token。
- [ ] 夜间默认深灰 + 柔和浅灰文字，不是纯黑纯白。
- [ ] 字号、行高、边距、背景修改实时生效。
- [ ] 目录、设置、AI 面板均为临时辅助层。
- [ ] 进度保存使用章节 + offset / anchor + percentage，不用页码作为真实锚点。

## C. 导入解析验收

- [ ] TXT 支持 UTF-8、GBK/GB18030、Big5 候选解码。
- [ ] TXT 章节识别使用候选评分，不只靠单条正则。
- [ ] 解析预览可编辑章节标题、删除误识别章节。
- [ ] URL 解析必须有失败提示、重试和手动修正路径。
- [ ] 解析失败不应破坏原有书籍数据。

## D. 响应式验收

- [ ] 340px 极窄屏下按钮不挤压，底部浮层不遮挡。
- [ ] 手机安全区正常。
- [ ] 桌面三栏不被强行压扁。
- [ ] 宽屏正文不铺满。
- [ ] 旋转屏幕后阅读位置可恢复。

## E. 代码架构验收

- [ ] `reader-core` 不依赖 React / Dexie / API。
- [ ] `parser-core` 不依赖 UI。
- [ ] `shared-types` 不依赖实现。
- [ ] 书源、搜索、AI、存储、队列均通过接口抽象。
- [ ] 新端复用核心包，不重写阅读器。
