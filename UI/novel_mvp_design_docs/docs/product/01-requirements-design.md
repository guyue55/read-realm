# 小说平台需求设计文档 v0.1（MVP 版）

## 1. 产品定位

本项目定位为：

> **个人小说聚合阅读平台：支持本地上传、链接解析、免费搜索发现、章节净化、跨端沉浸式阅读，并预留 AI 阅读增强能力。**

它不是单纯的电子书阅读器，也不是小说内容站。平台重点不是“提供内容”，而是帮助用户把已有文本、文件、网页链接和免费搜索发现结果整理成可持续阅读、可离线、可恢复、可修正的个人阅读资产。

## 2. MVP 核心目标

MVP 不追求完整阅读生态，只验证三个核心命题：

| 目标 | 说明 | 验收标准 |
|---|---|---|
| 能导入 | 用户能导入 TXT / EPUB，也能尝试粘贴 URL | 文件能成功进入解析流程，URL 能给出成功或可解释失败 |
| 能解析 | 系统能识别目录、章节、正文，并允许用户修正 | 解析预览可查看章节，支持修正章节标题 / 删除误识别章节 |
| 能舒服读 | 阅读页排版稳定、进度可恢复、离线可读 | 刷新后恢复阅读位置，已缓存章节无网可读 |

第一版北极星指标建议定义为：

> **用户能否顺畅完成：导入一本小说 → 自动解析章节 → 舒服阅读 → 下次准确续读。**

## 3. 目标用户

| 用户类型 | 核心需求 | 典型问题 |
|---|---|---|
| 本地藏书用户 | 上传 TXT/EPUB，自动切章，离线阅读 | TXT 编码乱码、章节识别错误、目录不准 |
| 网文追更用户 | 粘贴链接、解析网页、保存章节、后续阅读 | 网页广告多、目录结构乱、分页正文 |
| 长篇深读用户 | 稳定排版、前情摘要、书签、进度恢复 | 长篇容易忘前情、跳章后迷失 |
| 多端用户 | 手机浏览器、桌面浏览器、PWA 间续读 | 设备切换后进度丢失 |
| 技术型用户 | 自定义规则、导入导出、私有化部署 | 不想依赖收费平台或封闭生态 |

## 4. 核心用户场景

### 场景 A：上传 TXT 并阅读

```text
用户进入书架
  -> 点击导入
  -> 选择 TXT 文件
  -> 系统识别编码
  -> 系统识别章节
  -> 展示解析预览
  -> 用户检查目录
  -> 必要时修正章节标题
  -> 加入书架
  -> 打开阅读器
  -> 自动保存进度
  -> 下次恢复阅读
```

### 场景 B：上传 EPUB 并阅读

```text
用户选择 EPUB
  -> 系统读取 EPUB 元数据
  -> 解析 OPF / nav / NCX / spine
  -> 提取章节
  -> 转换为统一 Book / Chapter 结构
  -> 加入书架
  -> 使用统一阅读器阅读
```

### 场景 C：粘贴小说链接

```text
用户粘贴 URL
  -> 系统拉取页面
  -> 判断页面类型
  -> 识别详情页 / 目录页 / 章节页 / 普通文章页
  -> 提取书名、作者、目录、正文
  -> 清理正文
  -> 生成质量提示
  -> 用户选择临时阅读或加入书架
```

### 场景 D：继续阅读

```text
用户打开应用
  -> 书架显示继续阅读卡片
  -> 点击继续阅读
  -> 系统恢复上次章节和位置
  -> 若本地缓存存在，直接离线打开
  -> 若缓存缺失，提示重新导入或同步
```

## 5. MVP 范围

### 5.1 P0 必做

| 模块 | 功能 |
|---|---|
| 书架 | 书籍列表、继续阅读、导入入口、最近阅读 |
| 文件导入 | TXT 上传、EPUB 上传 |
| TXT 解析 | 编码识别、章节识别、目录识别、正文切分 |
| EPUB 解析 | 元数据、目录、章节内容提取 |
| 解析预览 | 章节列表、正文预览、质量提示、手动修正 |
| 链接阅读 | 粘贴 URL，尝试解析正文，临时阅读或加入书架 |
| 阅读器 | 滚动、分页、字号、行高、段距、边距、背景、夜间模式 |
| 目录 | 章节列表、当前章节定位、上一章、下一章 |
| 进度 | 自动保存阅读位置，打开后恢复 |
| 离线 | IndexedDB / OPFS / Cache Storage 缓存 |
| 书签 | 添加 / 取消当前章节或当前位置书签 |
| 搜索 | 书架搜索、章节标题搜索、基础正文片段搜索 |
| 设置 | 阅读设置、本地存储、API 健康检查 |
| AI | 当前章摘要 |
| 同步 | 前端 IndexedDB 与本地 API 轻量交换数据 |

### 5.2 P1 下一阶段

| 模块 | 功能 |
|---|---|
| 章节质量 | 缺章、重复章、断章、广告章、乱码检测 |
| 净化 | 广告 / 水印清理、空行压缩、分页合并 |
| 高亮笔记 | 文本高亮、笔记、批注列表 |
| 书内搜索 | 完整全文搜索、搜索结果定位 |
| AI 增强 | 前情提要、人物关系、设定 / 术语表 |
| 同步增强 | 账号登录、书架同步、进度同步、设置同步 |
| 书源规则 | 书源规则协议 v1、搜索引擎候选发现 |

### 5.3 P2 后续能力

| 模块 | 功能 |
|---|---|
| 多来源 | 换源、单章补源、自动更新检测 |
| 格式扩展 | PDF、DOCX、MOBI、AZW3 |

| 多端 | Android / iOS、Tauri 桌面端、微信小程序轻阅读端 |
| 创作 | 写作工作台、人物卡、大纲、章节细纲、节奏评分 |

## 6. MVP 不做清单

| 暂不做 | 原因 |
|---|---|
| 完整书源生态 | 先预留规则系统，MVP 不开放复杂运行时 |
| 多来源换源 | 先保证单来源解析和阅读稳定 |
| 高级批注系统 | 先做书签，笔记 / 高亮 P1 |
| 完整全文搜索 | P0 做基础搜索，P1 做完整书内全文搜索 |
| PDF 小说化 | PDF 不适合强行切章，后续固定版式阅读 |
| DOCX / MOBI / AZW3 | 预留转换接口，MVP 不优先 |
| 小程序 / 原生 App / 桌面端 | 先完成 Web/PWA 跨屏 MVP |
| 默认云端保存全文 | 存储压力大，先本地为主 |
| Playwright 默认动态渲染 | 资源重，MVP 只做最后兜底或预留接口 |

## 7. MVP 成功标准

| 指标 | 成功标准 |
|---|---|
| TXT 导入成功率 | 常见 UTF-8 / GB18030 / Big5 TXT 可导入 |
| EPUB 导入成功率 | 标准 EPUB 可读取目录和章节 |
| 阅读恢复 | 刷新后能回到上次章节和大致位置 |
| 离线能力 | 已缓存书籍离线可打开 |
| 移动体验 | 手机浏览器可完成导入、阅读、设置 |
| 桌面体验 | 桌面正文限宽，不被宽屏拉满 |
| 解析失败体验 | 失败原因可解释，可重试或手动处理 |
| 本地数据 | 书架、进度、设置、书签不丢失 |
| AI 摘要 | 当前章可生成或给出明确失败原因 |
| 同步 | 前端与本地 API 可导入 / 导出基础数据 |
