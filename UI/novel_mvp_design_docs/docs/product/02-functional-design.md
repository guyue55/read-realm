## F01 书架与书籍管理

### 功能目标

书架是用户长期使用的主入口，需要承载最近阅读、阅读进度、本地 / 在线来源标识、导入入口、搜索、书籍详情和离线状态。

### 功能点

| 功能 | 优先级 | 说明 |
|---|---:|---|
| 书籍列表 | P0 | 显示所有入库书籍 |
| 继续阅读 | P0 | 进入最近阅读位置 |
| 最近阅读 | P0 | 按最近打开排序 |
| 来源标识 | P0 | upload / url / search / manual |
| 阅读进度 | P0 | 当前章、百分比、上次阅读时间 |
| 书籍详情 | P0 | 查看目录、进度、书签、元数据 |
| 删除书籍 | P0 | 删除书籍、章节、进度、书签 |
| 书架搜索 | P0 | 按书名、作者、章节标题 |
| 分类标签 | P1 | 玄幻、都市、待看、追更中 |
| 元数据编辑 | P1 | 书名、作者、封面、简介 |
| 导入导出 | P1/P2 | 本地备份、云同步 |

### 状态

| 状态 | 说明 |
|---|---|
| 空书架 | 引导导入 TXT / EPUB 或粘贴 URL |
| 有书籍 | 展示继续阅读卡和书籍列表 |
| 本地缓存完整 | 显示已离线 |
| 仅元数据 | 提示正文未缓存 |
| 删除中 | 禁用操作，显示处理中 |
| 删除失败 | 显示原因，可重试 |
| 弱网 / 离线 | 仅显示已缓存可读内容 |

## F02 文件导入

### 功能目标

支持用户上传本地小说文件，MVP 优先支持 TXT / EPUB。文件导入优先在前端处理，因为文件已在用户本地，隐私好、速度快，且可降低后端压力。

### 支持格式

| 格式 | MVP | 处理方式 |
|---|---:|---|
| TXT | P0 | 前端解析优先 |
| EPUB | P0 | 前端解析优先 |
| HTML / MHTML | P1 | 前端尝试，后端兜底 |
| Markdown | P1 | 标题层级转章节 |
| PDF | P2 | PDF.js 固定版式 |
| DOCX | P2 | 后端转换 |
| MOBI / AZW3 | P2 | 后端转换 |
| CBZ / CBR | P2 | 漫画 / 图文阅读 |

### 导入流程

```text
选择文件
  -> 校验格式
  -> 读取文件
  -> 判断文件类型
  -> 进入对应解析器
  -> 生成 Book + Chapter[]
  -> 展示解析预览
  -> 用户确认
  -> 写入本地书架
```

### 异常

| 异常 | 处理 |
|---|---|
| 格式不支持 | 提示当前支持 TXT / EPUB |
| 文件过大 | 提示分片解析，避免卡死 |
| 文件读取失败 | 提示重新选择 |
| 浏览器存储不可用 | 提示可能无法离线 |
| 解析结果为空 | 禁止入库，提示重新解析 |
| 章节为空 | 回退为单章或提示手动处理 |

## F03 文本解析

### 统一内部结构

```ts
type Book = {
  id: string
  title: string
  author?: string
  sourceType: 'upload' | 'url' | 'search' | 'manual'
  format: 'txt' | 'epub' | 'html' | 'md' | 'pdf'
  cover?: string
  metadata: Record<string, any>
  chapters: Chapter[]
}

type Chapter = {
  id: string
  bookId: string
  index: number
  title: string
  content: string
  wordCount: number
  sourceUrl?: string
  hash: string
}
```

### TXT 解析规则

编码候选：UTF-8、GBK、GB18030、Big5。

章节标题模式：第1章、第一章、第001章、正文 第一章、卷一 第一章、Chapter 1、楔子、序章、终章、后记、番外一。

章节候选评分：

```text
章节候选行评分 =
  是否短行
+ 是否包含 第X章 / 第X节 / 第X回 / 卷X / Chapter X
+ 是否前后为空行
+ 是否出现在合理位置
+ 是否编号连续
- 是否过长
- 是否包含句号、逗号过多
- 是否像正文叙述
- 是否含广告词
```

### EPUB 解析规则

```text
读取 ZIP
  -> 读取 META-INF/container.xml
  -> 定位 OPF
  -> 解析 metadata / manifest / spine
  -> 解析 nav / NCX
  -> 按 spine 提取 XHTML
  -> 清理 HTML
  -> 转成 Chapter[]
```

## F05 URL 解析

### URL 解析优先级

```text
1. 普通 fetch + HTML 解析
2. Readability 正文提取兜底
3. 小说结构启发式解析
4. 书源规则 / 站点适配器
5. 轻量 JS 执行方案，可选
6. Playwright 动态渲染，最后兜底
```

### 页面类型

| 类型 | 识别依据 | 输出 |
|---|---|---|
| 详情页 | 书名、作者、简介、目录入口 | Book metadata |
| 目录页 | 大量章节链接 | ChapterMeta[] |
| 章节页 | 正文长、上一章 / 下一章 | Chapter |
| 普通文章页 | 无章节关系 | 临时阅读内容 |
| 未识别 | 结构不足 | 失败原因 |

### 安全规则

| 规则 | 说明 |
|---|---|
| 只允许 http / https | 禁止 file、ftp 等协议 |
| 阻止 localhost | 防 SSRF |
| 阻止私网 IP | 防访问内网 |
| 阻止 IPv6 loopback | 包括 IPv4-mapped IPv6 |
| 限制跳转 | 防恶意重定向 |
| 限制响应大小 | 防大文件拖垮 |
| 超时控制 | 防卡死 |

## F07 阅读进度

### 进度模型

```ts
type ReadingProgress = {
  bookId: string
  chapterId: string
  chapterIndex: number
  offset: number
  percentage: number
  anchor?: {
    type: 'textQuote' | 'paragraph' | 'cfi'
    quote?: string
    paragraphIndex?: number
    cfi?: string
  }
  updatedAt: string
}
```

### 保存策略

| 场景 | 策略 |
|---|---|
| 滚动中 | 节流保存 |
| 翻页后 | 立即保存 |
| 切章 | 立即保存 |
| 关闭页面 | 尝试最后保存 |
| 设置变化 | 不改变真实进度，只重定位 |
| 章节变化 | 尝试 anchor 恢复，失败用 percentage |
