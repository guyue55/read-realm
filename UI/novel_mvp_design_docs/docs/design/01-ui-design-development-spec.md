# 小说平台 UI 设计开发说明 v0.1

## 1. UI 总定位

本项目 UI 不是普通资讯站、商城、论坛或后台，而是 **个人小说聚合阅读平台**。UI 的核心目标是：

1. 用户能持续舒服地读。
2. 用户能快速恢复阅读上下文。
3. 用户能处理导入、链接解析、章节质量等小说来源问题。

阅读界面遵循 **低存在感、低干扰、低疲劳、高可控、强恢复** 的原则，并拆成三层：沉浸阅读层、操作控制层、内容增强层。沉浸阅读层负责正文排版和翻页；操作控制层负责目录、进度、设置、搜索、书签；内容增强层负责章节质量、净化、AI 摘要等能力。

MVP 目标仍然聚焦“能导入、能解析、能舒服读”，P0 范围包括书架、TXT/EPUB 导入、解析预览、阅读器、目录、进度、离线、书签、URL 解析和 AI 当前章摘要。

# 3. 页面详细设计

## 3.1 书架页 `/library`

### 页面目标

作为默认入口，让用户快速完成：

- 继续上次阅读。
- 查看全部书籍。
- 导入新小说。
- 搜索本地书架。
- 判断哪些书可离线阅读。
- 进入书籍详情或阅读页。

### 移动端结构

| 区域 | 内容 |
|---|---|
| 顶部栏 | 应用名、搜索入口、导入按钮、更多 |
| 继续阅读卡 | 封面、书名、当前章节、阅读进度、上次阅读时间、继续阅读按钮 |
| 快捷入口 | 本地导入、粘贴链接、最近阅读、离线缓存 |
| 书架列表 | 网格 / 列表、分类、排序、来源标签 |
| 底部导航 | 书架、发现/搜索、导入、我的/设置；阅读页不显示底部主导航 |

### 桌面端结构

| 区域 | 内容 |
|---|---|
| 左侧栏 | 全部、本地、URL、最近、已完结、设置入口 |
| 顶部工具区 | 搜索框、导入按钮、视图切换、排序 |
| 主内容区 | 继续阅读横向卡、书籍网格 |
| 右侧信息区，可选 | 最近导入、缓存状态、解析任务 |

### 功能

| 功能 | 优先级 | 说明 |
|---|---:|---|
| 继续阅读 | P0 | 点击进入上次章节和位置 |
| 书籍网格 | P0 | 显示封面、书名、进度、来源 |
| 书籍列表 | P1 | 信息密度更高 |
| 书架搜索 | P0 | 按书名、作者、章节标题 |
| 导入入口 | P0 | 跳转导入页 |
| 删除书籍 | P0 | 二次确认 |
| 分类 / 标签 | P1 | 后续增强 |

### 状态

| 状态 | UI 表现 |
|---|---|
| 空书架 | 柔和空状态 + “导入 TXT/EPUB” + “粘贴链接” |
| 有书 | 继续阅读卡 + 书籍网格 |
| 有解析任务 | 顶部任务条或卡片 |
| 书籍缓存完整 | 显示“已离线” |
| 书籍仅元数据 | 显示“未缓存正文” |
| 弱网 | 顶部弱提示：“当前离线，仅显示已缓存书籍” |
| 删除确认 | Bottom Sheet / Dialog，说明会删除本地进度和书签 |

## 3.4 导入页 `/import`

### 页面目标

作为内容进入平台的中控台。

### 移动端结构

- 标题：导入小说。
- 主卡片：上传 TXT/EPUB。
- 次卡片：粘贴小说链接。
- 辅助入口：粘贴正文，后续 P1。
- 最近任务：解析中、失败、可继续。
- 隐私说明：文件优先本地解析。

### 桌面端结构

- 左侧大拖拽上传区。
- 右侧 URL 输入卡。
- 底部支持格式说明。
- 最近导入任务列表。

### 功能

| 功能 | 优先级 | 说明 |
|---|---:|---|
| 上传 TXT | P0 | 支持 UTF-8、GB18030、Big5 等候选编码 |
| 上传 EPUB | P0 | 解析 OPF、nav/NCX、spine |
| 拖拽上传 | P0 桌面 | 桌面体验增强 |
| 粘贴 URL | P0 | 进入 URL 解析流程 |
| 格式不支持提示 | P0 | 明确说明 P0 支持 TXT/EPUB |
| 最近任务恢复 | P1 | 解析中断后继续 |

### 状态

| 状态 | UI |
|---|---|
| 初始 | 上传入口 + URL 入口 |
| 拖拽悬停 | 上传区高亮 |
| 文件过大 | 提示大文件会分片解析 |
| 格式不支持 | 说明支持范围 |
| 浏览器存储不可用 | 提示可能无法离线 |
| 无网络 | URL 入口弱化，文件入口可用 |

## 3.5 解析中页 / 解析任务状态

### 页面目标

让用户知道系统正在做什么，并能理解失败原因。

### 状态机

导入解析至少拆成：

1. 读取文件。
2. 识别编码。
3. 解析元数据。
4. 识别章节。
5. 清理正文。
6. 构建目录。
7. 生成预览。

文档中也明确导入、URL、AI、任务队列都不应该只用几个 boolean，而应使用状态机，方便失败恢复、进度展示、取消和错误定位。

### UI 元素

| 元素 | 说明 |
|---|---|
| 进度条 | 当前整体进度 |
| 阶段文案 | “正在识别章节标题” |
| 已识别数据 | 章节数、字数、疑似异常 |
| 取消按钮 | 允许停止解析 |
| 后台继续 | P1，可返回书架 |

### 状态

| 状态 | UI |
|---|---|
| 正常解析 | 阶段进度 + 已识别章节 |
| 大文件 | 显示“正在分片处理” |
| 编码不确定 | 弹出编码选择 |
| 章节识别弱 | 提示可能需要手动修正 |
| 解析失败 | 失败原因 + 重试 + 手动处理 |
| 用户取消 | 返回导入页，任务标记取消 |

## 3.7 URL 解析页 `/url` 或 `/import/url`

### 页面目标

用户粘贴小说网页链接后，系统尝试识别页面类型并提取正文/目录。

### 页面结构

| 区域 | 内容 |
|---|---|
| 输入区 | URL 输入框、解析按钮 |
| 解析说明 | 免费搜索/链接仅做发现，内容可用性不承诺 |
| 结果区 | 页面类型、标题、目录数量、正文长度、质量评分 |
| 操作区 | 临时阅读、加入书架、重试、手动提取 |
| 历史区 | 最近解析链接 |

### 页面类型

| 类型 | UI 表现 |
|---|---|
| 详情页 | 展示书名、作者、简介、目录入口 |
| 目录页 | 展示章节链接数量和前若干章 |
| 章节页 | 展示章节标题、正文预览、上一章/下一章 |
| 普通文章页 | 只允许临时阅读或作为单章导入 |
| 未识别 | 给出原因和手动处理入口 |

### 失败原因必须可解释

阅读功能调研明确指出，链接解析失败时要说明是 CORS、动态渲染、未识别目录、正文过短还是疑似反爬，并允许手动选择正文区域或目录区域。

### 状态

| 状态 | UI |
|---|---|
| 初始 | URL 输入框 |
| 解析中 | 阶段进度：拉取页面、识别类型、提取正文 |
| 成功 | 质量评分 + 导入入口 |
| 正文过短 | 提示可能不是章节页 |
| 目录为空 | 允许作为单章临时阅读 |
| CORS / 网络失败 | 提示后端兜底或稍后重试 |
| 疑似动态页面 | 提示 MVP 不默认启用重型动态渲染 |
| URL 安全拦截 | 提示不支持 localhost、私网地址等 |

## 3.8 阅读页 `/reader/:bookId/:chapterIndex?`

### 页面目标

这是产品核心。默认状态必须让用户忘记 UI，只读正文。

### 阅读页主状态

| 状态 | 触发 | 表现 |
|---|---|---|
| 沉浸态 | 默认进入 | 只显示正文和极弱进度 |
| 菜单态 | 点击中央区域 | 顶部栏 + 底部栏 |
| 目录态 | 点击目录 | 目录抽屉 |
| 设置态 | 点击设置 | 设置面板 |
| 进度态 | 点击进度 | 进度面板 |
| AI 态 | 点击 AI | AI 摘要面板 |
| 离线态 | 网络不可用 | 保持已缓存章节可读 |
| 加载态 | 切章 / 打开 | 骨架或轻量 loading |
| 错误态 | 章节缺失 | 说明原因 + 返回目录 |

阅读 UI 已明确要求动效克制，正文滚动和分页不能做复杂炫动，只允许菜单、抽屉、设置面板有轻微过渡。

### 移动端沉浸态

| 区域 | 设计 |
|---|---|
| 顶部 | 可选极弱章节名 |
| 正文 | 单栏、纸感背景、左右安全边距 |
| 底部 | 极弱当前章进度 / 全书进度 |
| 工具栏 | 默认隐藏 |
| 主导航 | 不显示底部主导航 |

### 移动端点击区

| 区域 | 行为 |
|---|---|
| 左 25% | 上一页 / 上一屏 |
| 中 50% | 呼出菜单 |
| 右 25% | 下一页 / 下一屏 |

移动端首发需要 Safe Area、触控手势、点击区翻页、底部菜单、半屏设置面板，并且手机浏览器体验本身就是 P0。

### 桌面端阅读布局

| 模式 | 说明 |
|---|---|
| 单栏沉浸 | 正文居中，最大宽度 680–820px |
| 左目录 + 正文 | 左侧目录，正文居中 |
| 正文 + AI/笔记 | 右侧辅助面板 |
| 三栏工作区 | 目录 / 正文 / AI 笔记 |
| 宽屏双栏 | 每栏 360–460px，P1 |

桌面端不得让正文铺满宽屏；正文最大宽度和三栏固定比例是开发硬约束。已有设计中也专门修正过桌面工作区挤压问题，采用固定 1280px 画布和稳定三栏比例，避免窄屏强行压扁。

## 3.9 阅读菜单态

### 移动端顶部栏

## 3.16 离线 / 弱网页

### 页面目标

保证用户知道哪些内容还可读，哪些需要联网。

### 场景

| 场景 | UI |
|---|---|
| App Shell 离线打开 | 显示书架中已缓存书籍 |
| 当前书已缓存 | 允许继续阅读 |
| 当前章节未缓存 | 提示“此章节未缓存” |
| URL 解析离线 | 禁用解析按钮 |
| AI 离线 | 有缓存则展示，无缓存则提示联网 |
| API 不可用 | 本地阅读不受影响 |

### 关键要求

- 不要把离线当成错误页。
- 阅读过的内容优先可读。
- 未缓存内容明确说明恢复方式。

## 3.17 解析失败页

### 页面目标

失败可解释、可恢复。

### 失败类型

| 类型 | 文案方向 | 操作 |
|---|---|---|
| 文件格式不支持 | 当前仅支持 TXT/EPUB | 更换文件 |
| 编码失败 | 无法识别文本编码 | 手动选择编码 |
| 章节识别失败 | 未识别到目录，将作为单章导入 | 单章导入 / 重新识别 |
| 正文为空 | 未提取到有效正文 | 重新解析 |
| URL 网络失败 | 网络或目标站不可用 | 重试 |
| URL 安全拦截 | 不支持私网或本地地址 | 更换链接 |
| 疑似动态页面 | 当前 MVP 不默认动态渲染 | 复制正文 / 手动导入 |

# 5. 响应式规则

## 5.1 断点

| Viewport | 建议范围 | 阅读布局 |
|---|---|---|
| mobile | `< 768px` | 单栏、底部菜单、点击区翻页 |
| tablet | `768–1023px` | 单栏 / 双栏可选、抽屉目录 |
| desktop | `1024–1439px` | 单栏居中、可开侧栏 |
| wide | `>= 1440px` | 三栏工作区、双栏阅读可选 |

## 5.2 移动端硬约束

- 手机浏览器完整可用。
- PWA 可安装。
- 支持 Safe Area。
- 阅读页不显示底部主导航。
- 阅读菜单、设置、目录使用临时浮层。
- 底部栏高度必须包含 `env(safe-area-inset-bottom)`。
- 输入场景要考虑软键盘遮挡。

## 5.3 桌面端硬约束

- 正文不能铺满宽屏。
- 三栏工作区不能被强行压扁。
- 窄屏可横向滚动或退化为单栏。
- AI/笔记不能抢正文优先级。
- 鼠标和键盘快捷键可作为增强，不影响移动端主链路。

# 8. 当前结论

后续结合设计图开发时，建议以这 18 个 P0 页面/状态为第一版开发边界：

```text id="i51hyj"
1. 书架页
2. 空书架页
3. 书籍详情页
4. 导入页
5. 解析中状态
6. 解析预览页
7. URL 解析页
8. 阅读页沉浸态
9. 阅读页菜单态
10. 目录抽屉
11. 阅读设置面板
12. 阅读进度面板
13. AI 摘要面板
14. 搜索页
15. 设置页
16. 离线/弱网页
17. 解析失败页
18. 同步/本地数据状态页
```

其中真正的 MVP 主链路是：

```text id="65xdvc"
书架
  -> 导入 TXT/EPUB 或粘贴 URL
  -> 解析中
  -> 解析预览
  -> 加入书架
  -> 阅读页
  -> 目录 / 设置 / 进度 / 书签 / AI
  -> 下次恢复阅读
```

这条链路跑通并稳定后，再进入 P1 的笔记、高亮、书内全文搜索、章节质量、书源管理和多端同步。
